02

Port Security

Close everything. Only open what is absolutely necessary and restrict by IP.

Default Deny All Incoming

UFW (Uncomplicated Firewall) must be configured to deny all incoming connections by default. Explicitly allow only required ports from specific IPs.

Port Service Required Configuration
22 SSH YES Key auth only, restrict to office IP
3000 OpenClaw Gateway YES Your home IP only, VPN recommended
443 HTTPS OPTIONAL If web interface public
3306 MySQL NO NEVER expose. Localhost only.
5432 PostgreSQL NO NEVER expose. Bind 127.0.0.1.
6379 Redis NO NEVER expose. No auth by default.
UFW Firewall Configuration
# Install UFW apt install ufw -y # Default deny all incoming ufw default deny incoming ufw default allow outgoing # Allow SSH from specific IP only ufw allow from YOUR_OFFICE_IP to any port 22 # Allow OpenClaw from your IP only ufw allow from YOUR_HOME_IP to any port 3000 # Explicitly block dangerous ports ufw deny 3306/tcp # MySQL ufw deny 5432/tcp # PostgreSQL ufw deny 6379/tcp # Redis ufw deny 27017/tcp # MongoDB # Enable and verify ufw enable ufw status verbose
03

SSH Hardening

SSH is the #1 attack vector. Disable passwords, use keys, install fail2ban.

01

Disable Password Authentication

Edit /etc/ssh/sshd_config: Set PasswordAuthentication no, PermitRootLogin no, PubkeyAuthentication yes. Restart sshd service.

02

Change SSH Port (Optional)

Move from default port 22 to non-standard port (e.g., 2222). Update UFW rules. Reduces automated scan noise by 99%.

03

Install Fail2Ban

Auto-ban IPs after 3 failed attempts. Bantime: 1 hour. Reduces brute force by blocking attackers automatically.

Fail2Ban Configuration
# Install apt install fail2ban -y # Create config cat > /etc/fail2ban/jail.local << 'EOF' [DEFAULT] bantime = 3600 findtime = 600 maxretry = 3 [sshd] enabled = true port = ssh filter = sshd logpath = /var/log/auth.log maxretry = 3 EOF # Enable systemctl enable fail2ban systemctl start fail2ban fail2ban-client status sshd
04

API Token Protection

Leaked tokens are exploited within minutes. Never commit to git. Restrict permissions.

Immediate Revocation Required

If you accidentally commit an API key to git, revoke it immediately in your provider dashboard (Anthropic, OpenAI, Telegram). Bots scan GitHub in real-time.

Secure Token Storage
# Create secure directory mkdir -p /etc/openclaw chmod 700 /etc/openclaw # Write tokens cat > /etc/openclaw/.env << 'EOF' TELEGRAM_BOT_TOKEN=your_token ANTHROPIC_API_KEY=your_key EOF # Lock down: readable only by root chmod 600 /etc/openclaw/.env chown root:root /etc/openclaw/.env # Verify ls -la /etc/openclaw/ # -rw------- 1 root root
05

Pre-Deployment Checklist

Verify each item before going live. Non-negotiable for production.

UFW Firewall Enabled

Default deny incoming. Only ports 22 and 3000 allowed from specific IPs.

SSH Key Authentication Only

PasswordAuthentication disabled. PermitRootLogin set to no.

Fail2Ban Installed & Active

Ban after 3 failed attempts. Check status: fail2ban-client status sshd

.env File Permissions 600

Owned by root. Not readable by other users. Verified with ls -la.

No Database Ports Exposed

MySQL, PostgreSQL, Redis all bound to 127.0.0.1 only. UFW denies external access.

OpenClaw Gateway IP-Restricted

Port 3000 accessible only from your home IP or VPN range.

Telegram Bot User Restricted

Only your Telegram user ID in allowed_users list. No public access.

Automatic Updates Enabled

unattended-upgrades installed for security patches.

06

Emergency Response

If you suspect compromise, act immediately.

Signs of Compromise

Unexpected CPU usage (crypto mining), unknown processes, modified files, failed login attempts from unknown IPs, or unexpected network traffic.

01

Revoke All API Keys

Immediately revoke Telegram bot token, Anthropic/OpenAI keys. Generate new ones after cleanup.

02

Snapshot & Analyze

Create DigitalOcean snapshot for forensics. Do not destroy yet - analyze the attack vector.

03

Rebuild Clean

Destroy compromised droplet. Deploy fresh instance with security hardening from step 1.
