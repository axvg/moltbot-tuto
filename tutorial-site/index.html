2 Create Your Telegram Bot

Your Telegram bot will be the control interface for your AI developer. Here's how to create it:

Open Telegram and Find @BotFather

Search for "BotFather" in Telegram (it's the official bot for creating bots). Start a chat and type: /newbot

Name Your Bot

Choose a display name (e.g., "My AI Developer") and then a username that must end in "bot" (e.g., "myai_dev_bot")

Save the API Token

BotFather will give you a token like: 123456789:ABCdefGHIjklMNOpqrSTUvwxyz. Save this - you'll need it later!

Get Your User ID

Message @userinfobot on Telegram. It will reply with your user ID (a number like 123456789). Save this too.

Tokens to Save 
TELEGRAM_BOT_TOKEN: 123456789:ABCdefGHIjklMNOpqrSTUvwxyz
YOUR_USER_ID: 123456789

3 Setup DigitalOcean Droplet

Now let's create your server that will run Moltbot 24/7. This is easier than you think!

Option A: 1-Click Marketplace (Easiest)

Go to DigitalOcean Marketplace

Visit marketplace.digitalocean.com/apps/moltbot and click "Create Moltbot Droplet"

Choose Your Plan

Select "Basic" plan with at least 2GB RAM ($12/month recommended for production use)

Select Datacenter

Pick a region close to you (e.g., New York, London, Singapore)

Authentication

Choose "SSH Key" (recommended) or password. Add your SSH key if you have one.

Create and Wait

Click "Create Droplet". Wait 2-3 minutes for it to be ready. You'll receive an email with the IP address.

4 Configure Moltbot

Now let's configure Moltbot to be your personal freelance coder. We'll set up coding capabilities, task management, and security.

Step 1: SSH Into Your Server

bash 
ssh root@YOUR_DROPLET_IP

Step 2: Create Configuration Directory

bash 
# Create directories
mkdir -p /etc/moltbot
mkdir -p /home/moltbot/projects
mkdir -p /home/moltbot/notes
mkdir -p /home/moltbot/data

# Create moltbot user
useradd -m -s /bin/bash moltbot
chown -R moltbot:moltbot /home/moltbot

Step 3: Set Up Environment Variables

bash 
# Create .env file
nano /etc/moltbot/.env

# Add these lines (replace with your actual values):
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here
YOUR_TELEGRAM_USER_ID=your_telegram_user_id
ANTHROPIC_API_KEY=your_anthropic_api_key_here
‚ö†Ô∏è
Security Warning

Never share your .env file or commit it to git! These are your private API keys. The .env file should have permissions 600 (read/write only by owner).

Step 4: Create Main Configuration

Save this comprehensive configuration to /etc/moltbot/config.yaml:

yaml 
telegram:
  enabled: true
  bot_token: "${TELEGRAM_BOT_TOKEN}"
  allowed_users:
    - ${YOUR_TELEGRAM_USER_ID}

ai:
  provider: anthropic
  api_key: "${ANTHROPIC_API_KEY}"
  model: "claude-3-5-sonnet-20241022"

developer_mode:
  enabled: true
  auto_save_code: true
  code_directory: "/home/moltbot/projects"
  preferred_languages:
    - python
    - javascript
    - typescript
    - go
    - bash

  enabled: true
  storage_path: "/home/moltbot/data/tasks"

  enabled: true
  allowed_paths:
    - "/home/moltbot/projects"
    - "/home/moltbot/notes"
    - "/tmp"
  blocked_paths:
    - "/etc"

    - "/usr/bin"

5 Install MCP Servers

MCP (Model Context Protocol) servers give your bot superpowers - browsers, terminal access, file system, and 8000+ app integrations!

Install Required Dependencies

bash 
# Install Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt-get install -y nodejs

# Install MCP servers globally
npm install -g @executeautomation/playwright-mcp-server
npm install -g @modelcontextprotocol/server-filesystem
npm install -g @modelcontextprotocol/server-terminal

# Install Playwright browsers
npx playwright install firefox
npx playwright install chromium

Configure MCP Servers

Create /etc/moltbot/mcp.json:

json 
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@executeautomation/playwright-mcp-server"]
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/moltbot/projects"]
    },
    "terminal": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-terminal"]
    }
  }
}

Start Moltbot

bash 
# Start Moltbot service
systemctl start moltbot

# Enable auto-start
systemctl enable moltbot

# Check status
systemctl status moltbot

# View logs in real-time
journalctl -u moltbot -f

8 Troubleshooting

Running into issues? Here are common problems and solutions:

Bot Not Responding

bash 
# Check if service is running
systemctl status moltbot

# If not running, start it
systemctl start moltbot

# Check logs for errors
journalctl -u moltbot -n 50 --no-pager

# Test Telegram token
curl -s "https://api.telegram.org/botYOUR_TOKEN/getMe"

MCP Servers Not Working

bash 
# Check if Playwright is installed
npx playwright --version

# Reinstall browsers if needed
npx playwright install

# Check MCP servers
which npx
npm list -g @executeautomation/playwright-mcp-server

Permission Errors

bash 
# Fix ownership
chown -R moltbot:moltbot /home/moltbot
chown -R moltbot:moltbot /etc/moltbot

# Fix permissions on .env
chmod 600 /etc/moltbot/.env
